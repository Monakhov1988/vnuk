ТЗ на разработку MVP «Внучок» (Replit Edition)

1. Стек технологий
•   Язык: Python 3.10+
•   Библиотека бота: aiogram (асинхронная, современная).
•   База данных: Replit PostgreSQL (встроенная).
•   AI Мозги: OpenAI API (GPT-4o-mini — для логики, Whisper — для распознавания голоса).
•   Голос (TTS): Yandex SpeechKit или ElevenLabs (для «теплого» звучания).
•   Планировщик: APScheduler (для напоминаний о таблетках).

2. Структура Базы Данных (PostgreSQL)
Нужны следующие таблицы:
1.  Users: id, tg_id, role (parent/child), linked_id (связь родитель-ребенок), name, timezone.
2.  Reminders: id, user_id, APScheduler0, APScheduler1, APScheduler2, status (confirmed/missed).
3.  Memoirs: APScheduler4, user_id, audio_url, APScheduler7, APScheduler8.
4.  UtilityMetrics: id, user_id, photo_url, value, id3 (water/electricity), id4.

3. Функциональные модули

Модуль А: Регистрация и связка
•   Логика: Ребенок заходит в бота, получает уникальный код. Родитель заходит в бота, вводит код -> аккаунты связаны.
•   Интерфейс: Кнопки «Я родитель» / «Я ребенок».
Модуль Б: «Здоровье» (Таблетки)
•   Для ребенка: Интерфейс в боте для добавления напоминания (название, время).
•   Для родителя: В назначенное время бот присылает сообщение: «[Имя], пора принять [Лекарство]» + кнопка «Выпила».
•   Эскалация: Если кнопка не нажата в течение 20 минут — сообщение ребенку: «⚠️ Мама не подтвердила прием лекарств».

Модуль В: «Голосовой помощник» (STT + LLM)
•   Бот принимает аудиосообщения от родителя.
•   Процесс: Audio -> Whisper (текст) -> GPT (анализ намерения).
•   Намерения (Intents):
    •   «Хочу поболтать» -> GPT отвечает в добром стиле.
    •   «Мне плохо» -> Срочный алерт ребенку + автозвонок (через API).
    •   «Запиши историю» -> Переход в режим Мемуаров.

Модуль Г: «Счетчики» (Vision)
•   Родитель присылает фото счетчика.
•   Бот отправляет фото в OpenAI Vision API.
•   Результат: Бот пишет: «Я увидел цифры 123. Записать?». После «Да» данные улетают ребенку.

Модуль Д: «Вечерние мемуары»
•   Раз в день в 19:00 бот пишет родителю: «Расскажи, как прошел день? Или вспомни что-нибудь из детства».
•   Записывает аудио -> транскрибирует -> сохраняет в таблицу Memoirs.

4. Архитектура файлов в Replit
1.  main.py — точка входа, запуск бота.
2.  database.py — логика работы с PostgreSQL.
3.  scheduler.py — управление напоминаниями.
4.  ai_logic.py — интеграция с OpenAI (STT, Vision, Chat).
5.  keyboards.py — все кнопки интерфейса.
6.  utils.py — обработка аудио, форматирование текста.

5. Этапы реализации в Replit (Roadmap)
•   День 1: Создание проекта, настройка aiogram, база данных и простая регистрация.
•   День 2: Модуль таблеток и планировщик (APScheduler). Тесты алертов ребенку.
•   День 3: Подключение OpenAI (Голос + Текст). Режим простого общения.
•   День 4: Модуль счетчиков (Vision) и запись мемуаров.
•   День 5: Тестирование «в полях» на себе и своих родителях. Настройка Deployment в Replit.
